# PRD Rules

Create a PRD for every non-trivial product feature.

PRD path:

`docs/prds/{JIRA_ID}-{slug}.md`

A PRD describes what must be delivered and why.

A PRD must include:

- Jira ID
- Status
- Links
- Affected Projects
- Problem
- Goal
- Scope
- Out of Scope
- Functional Requirements
- Non-Functional Requirements
- Acceptance Criteria
- Risks
- Open Questions

Do not implement before PRD approval unless the user explicitly requests a quick implementation.
